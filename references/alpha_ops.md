# Alpha Expression Field & Function Contract

This document defines the exact contract for alpha expressions generated by this skill.
All generated expressions **must** use only the fields and functions listed below.

## Available Field Variables

Each field is a `date × symbol` DataFrame:

| Field    | Description              |
| -------- | ------------------------ |
| `open`   | Daily open price         |
| `high`   | Daily high price         |
| `low`    | Daily low price          |
| `close`  | Daily close price        |
| `volume` | Daily trading volume     |
| `amount` | Daily trading amount     |

> ⚠️ Only these 6 fields exist. Do **NOT** use `vwap`, `adjfactor`, or any other field.

## Available Functions

### Cross-Sectional (per date)

| Function          | Description                                    |
| ----------------- | ---------------------------------------------- |
| `rank(x)`         | Cross-sectional percentile rank at each date (0–1) |
| `zscore(x)`       | Cross-sectional z-score at each date           |
| `scale(x, a=1)`   | Cross-sectional rescaling so \|sum\| == a       |

### Time-Series Rolling

| Function                | Description                                          |
| ----------------------- | ---------------------------------------------------- |
| `ts_rank(x, n)`         | Time-series percentile rank over n periods           |
| `ts_zscore(x, n)`       | Rolling z-score over n periods                       |
| `ts_mean(x, n)`         | Rolling mean                                         |
| `ts_std(x, n)`          | Rolling standard deviation                           |
| `ts_max(x, n)`          | Rolling maximum                                      |
| `ts_min(x, n)`          | Rolling minimum                                      |
| `ts_sum(x, n)`          | Rolling sum                                          |
| `ts_argmax(x, n)`       | Periods since rolling maximum within the last n bars |
| `ts_argmin(x, n)`       | Periods since rolling minimum within the last n bars |
| `correlation(x, y, n)`  | Rolling Pearson correlation per symbol               |
| `covariance(x, y, n)`   | Rolling covariance per symbol                        |
| `decay_linear(x, n)`    | Linearly-decayed weighted moving average             |

### Lag / Difference

| Function        | Description              |
| --------------- | ------------------------ |
| `delay(x, n)`   | Lag x by n periods       |
| `delta(x, n)`   | x - delay(x, n)          |
| `returns(x, n=1)` | pct_change over n periods (default x=close) |

### Volume-Specific

| Function  | Description                       |
| --------- | --------------------------------- |
| `adv(n)`  | Rolling mean of volume over n periods |

### Element-Wise Math

| Function                           | Description                    |
| ---------------------------------- | ------------------------------ |
| `sign(x)`                          | Sign of x                      |
| `log(x)`                           | Natural logarithm              |
| `abs(x)`                           | Absolute value                 |
| `power(x, n)`                      | x raised to power n            |
| `signed_power(x, n)`               | sign(x) * abs(x)^n             |
| `min(x, y)`                        | Element-wise minimum           |
| `max(x, y)`                        | Element-wise maximum           |
| `clip(x, lower, upper)`            | Clip values to [lower, upper]  |

### Arithmetic

Standard arithmetic operators `+`, `-`, `*`, `/` work between DataFrames and scalars.

## Example Expressions

```
# Momentum: close vs close 5 days ago
returns(close, 5)

# Volume-weighted reversal
-1 * ts_rank(delta(close, 1), 5) * rank(volume / adv(20))

# Liquidity: turnover normalized
rank(volume / adv(20))

# Price position within range
(close - ts_min(low, 20)) / (ts_max(high, 20) - ts_min(low, 20))
```

## Validation Rules

1. Every function name must match exactly (case-sensitive).
2. Argument counts and types must match the signatures above.
3. Only `open`, `high`, `low`, `close`, `volume`, `amount` may appear as leaf variables.
4. No nested expressions that produce non-DataFrame intermediate values unless passed to a supported function.

## Look-Ahead Bias Prevention

> ⚠️ **Your alpha will be REJECTED if it peeks into the future.**

- `delay(x, n)`, `delta(x, n)`, and `returns(x, n)` **MUST** use `n ≥ 1`.
  Using `n = 0` or `n < 0` is explicit future-peeking → **REJECTED**.
- All rolling functions (`ts_mean`, `ts_rank`, `ts_std`, etc.) inherently
  use only past data — this is safe.
- Do **NOT** reference fields at relative offsets like `close[-1]` or
  `close[+1]`. Use `delay(close, 1)` instead of `close[-1]`.
- The validator runs a **truncation test**: it evaluates your expression on
  a dataset with the last 20% of dates removed and compares against the full
  evaluation. If results differ substantially in the overlapping region,
  your expression is flagged for look-ahead bias.

## Numerical Stability Rules

> ⚠️ **Your alpha will be REJECTED if it is numerically unstable.**

- **Never divide by something that could be zero.** Guard with:
  ```
  x / max(denom, 1e-8)       # safe division
  x / clip(denom, 1e-8, 1e8) # safe division with upper bound
  ```
- **Never take `log(x)` of non-positive values.** Guard with:
  ```
  log(max(x, 1e-8))          # safe log
  log(clip(x, 1e-8, 1e8))    # safe log with upper bound
  ```
- **Avoid overflow**: `power(x, 20)` on large x can explode.
  Prefer `signed_power(x, n)` for large exponents.
- **Prefer naturally stable expressions**: `rank()`, `zscore()`, `scale()`
  produce bounded, well-behaved outputs.
- The validator checks for:
  - NaN ratio > 50% → expression produces too many invalid values
  - Inf ratio > 1% → overflow or division by near-zero
  - Output std > 10× input scale → expression is numerically explosive
  - Zero-variance output → expression does not differentiate between symbols
